# 试读、代理与章节合并设计

## 目标

在现有客户端抓取、服务端解码架构上增加三项客户端能力：

1. 并行获取章节公开试读内容。
2. 使用用户提供的 HTTP/SOCKS 代理列表轮换请求。
3. 按起点目录顺序和卷信息生成单文件 TXT。

客户端继续只提取起点页面中的原始章节字段，并调用服务端
`POST /api/backup/{taskId}/decode-zip` 完成解码。现有 `.qd` 上传解密功能保留，
不在客户端增加任何解密算法、密钥计算或字体/CSS 解码实现。

## 架构

### `proxy.py`

提供线程安全的 `ProxyPool`：

- 接受用户显式配置的代理 URL 列表。
- 支持 `http://`、`https://`、`socks4://`、`socks5://`。
- 每处理指定数量的请求后切换代理。
- 未配置代理时返回空配置，所有请求保持直连。
- 不包含代理供应商地址、账号、密钥或自动购买逻辑。

### `chapter_preview.py`

负责公开试读：

- 独立 HTTP 请求获取移动端章节页。
- 从 `vite-plugin-ssr_pageContext` JSON 中读取 `chapterInfo.content`。
- 将 HTML 转成规范纯文本。
- 通过受控线程池批量获取试读。
- 用确定性的最长重叠算法将试读与服务端返回正文拼接，避免重复段落。

### `chapter_merge.py`

负责输出编排：

- 输入完整目录元数据和章节输出目录。
- 按目录顺序读取 `{chapterId}.txt`。
- 按卷插入卷标题。
- 可生成全书目录。
- 缺失章节保留标题和“本章未保存”标记。
- 输出到书籍目录下的 `{书名}.txt`。

### 现有流水线

`local_crawl_engine.local_crawl()` 新增可选参数：

- `preview_enabled`
- `proxy_urls`
- `proxy_rotate_every`
- `merge_enabled`
- `book_name`

获取试读与批次下载并行执行。服务端 ZIP 解码全部完成后，将试读合并进
`{chapterId}.txt`，最后生成全书合并文件。函数仍返回 `(success, failed)`，
保持现有调用兼容。

服务器抓取模式在“下载全部”完成后使用同一套试读与合并后处理。

## CLI

`backup` 新增：

- `--preview`：补全公开试读。
- `--merge`：生成单文件 TXT。
- `--proxy URL`：可重复指定，或使用逗号分隔代理。
- `--proxy-rotate-every N`：代理轮换间隔。

默认不启用代理。桌面端默认启用试读补全和合并。

## PyQt6

书籍详情页增加备份选项：

- “补全试读”复选框。
- “生成合并 TXT”复选框。
- 代理列表输入框，支持逗号分隔。

选项通过主窗口传递给备份面板。后台工作继续使用 `pyqtSignal` 更新界面，
不从工作线程直接操作控件。

## 安全边界

- 不复制 `qidian_local` 中硬编码的代理供应商 URL 和密钥。
- 不关闭 TLS 证书验证。
- 不实现服务端解码逻辑。
- 不修改服务端 API 契约，因此不需要跨仓库协调文档。
- 代理凭据仅存在于用户运行时输入或命令行参数，不写入仓库。
